# Документация Проекта

## О проекте

Проект: `GlobalStay`

Текущее состояние:
- настроен splash screen с логотипом;
- добавлен onboarding из 3 экранов;
- проект разложен по feature-структуре для дальнейшего роста.

## Структура `lib`

```text
lib/
  app/
    app.dart
  features/
    splash/
      presentation/
        splash_screen.dart
    onboarding/
      presentation/
        onboarding_screen.dart
        widgets/
          page_indicator.dart
          status_bar.dart
    home/
      presentation/
        home_screen.dart
  main.dart
```

## Назначение файлов

### `lib/main.dart`
- точка входа приложения;
- включает `SystemUiMode.immersiveSticky`;
- запускает `GlobalStayApp`.

### `lib/app/app.dart`
- корневой `MaterialApp`;
- хранит базовую тему приложения;
- задает стартовый экран.

### `lib/features/splash/presentation/splash_screen.dart`
- экран сплэша;
- показывает логотип;
- через короткую задержку переводит пользователя на onboarding.

### `lib/features/onboarding/presentation/onboarding_screen.dart`
- основной экран онбординга;
- управляет `PageView` только для карусели изображений;
- хранит контент 3 onboarding-экранов;
- держит текст, индикатор и кнопку поверх карусели без пролистывания;
- переключает пользователя по кнопке `Далее` / `Начать`.

### `lib/features/onboarding/presentation/widgets/page_indicator.dart`
- виджет пагинации;
- отображает активную страницу onboarding.

### `lib/features/onboarding/presentation/widgets/status_bar.dart`
- декоративный верхний status bar из дизайна;
- используется на onboarding-экранах как часть макета.

### `lib/features/home/presentation/home_screen.dart`
- главный экран после onboarding;
- содержит верхний поиск, кнопки фильтров/карты и список карточек жилья;
- данные карточек сейчас временные и находятся внутри экрана.

## Geoapify

Для карты используется Geoapify на базе OpenStreetMap.

API key хранится только локально в `.env.geoapify.json`.
Файл добавлен в `.gitignore`, передавать его в приложение нужно через `--dart-define-from-file=.env.geoapify.json`.

## Текущие UI-решения

### Splash
- белый фон;
- логотип по центру;
- переход на onboarding после задержки.

### Onboarding
- фоновые изображения берутся из assets;
- картинки растягиваются через `BoxFit.fill`;
- поверх картинки накладывается затемняющий градиент;
- верхняя строка статуса является частью дизайна;
- пагинация и кнопка выровнены под референс из Figma.

### Home
- шапка с поисковой строкой находится сверху на оранжевом фоне;
- нажатие на поисковую строку открывает нижний sheet `Снять жилье`;
- sheet содержит город, даты заезда/выезда, количество взрослых, детей и животных;
- даты заезда/выезда выбираются через системный календарь `showDatePicker`;
- выбранные критерии поиска сохраняются в состоянии `HomeScreen` для будущей фильтрации БД;
- кнопка фильтров открывает нижний sheet `Фильтры`;
- sheet фильтров содержит вид жилья, диапазон цен, удобства и дополнительные характеристики;
- выбранные фильтры сохраняются в состоянии `HomeScreen` для будущей фильтрации БД;
- кнопка карты переключает главный экран в режим карты Geoapify;
- карта подключена через `flutter_map`, API key передается через `--dart-define-from-file=.env.geoapify.json`;
- пины цен отрисованы через `MarkerLayer` и привязаны к координатам карточек;
- список жилья собран из временных карточек с фото, рейтингом, городом, типом жилья, датами и ценой;
- изображения карточек временно используют существующие assets onboarding.
- для списка и карты подготовлено пустое состояние `Нет подходящих вариантов`
  с иллюстрацией `docs/imgs/Empty State Illustrations_Light Mode_No Location Found.png`.

## Что важно помнить

- данные карточек на `HomeScreen` временные и позже должны прийти из поиска/API;
- тексты onboarding нужно хранить аккуратно в UTF-8 без сломанной кодировки;
- новые экраны лучше сразу добавлять в `features/<feature_name>/presentation/`.
