# Техническое задание: Сервис бронирования жилья

---

## Мобильное и веб-приложение для клиента

### Поиск жилья

Отображается список доступного жилья для бронирования. Можно переключиться в режим просмотра "карта". Жилье на карте отображаются маркерами. При клике на маркер открывается краткое описание жилья.

Пользователь может искать жилье, задавая:
- город
- дату заезда и дату выезда
- количество взрослых
- количество детей
- количество животных

Отображаются карточки жилья, соответствующие параметрам. На карточке жилья отображается краткая информация:
- вид жилья
- цена
- рейтинг
- фото

Пользователь может дополнительно отфильтровать результаты по следующим параметрам:
- Тип размещения
- Диапазон цен
- Удобства
- Доп характеристики (бассейн, завтраки, камин и т.д.)

Пользователь может перейти в карточку жилья. Отображается информация о жилье:
- фото
- адрес
- виды удобств
- описание
- отзывы
- стоимость

Отображается информация о владельце жилья:
- имя
- фото
- рейтинг/отзывы
- информация о владельце
- другие объявления владельца

Пользователь может добавить жилье в избранное. Пользователь может перейти к оформлению запроса на бронирование жилья:
- Забронировать жилье могут только авторизованные пользователи
- Предлагаем пользователю зарегистрироваться
- Пользователю отображается:
  - даты бронирования
  - количество гостей
  - общая сумма бронирования
  - сумму, которую нужно будет оплатить онлайн как предоплату
  - сумму, которую нужно будет заплатить непосредственно владельцу при заселении
  - условия отмены бронирования

Пользователь может написать владельцу. Пользователь может отправить запрос на бронирование жилья владельцу. Владелец жилья получает запрос на бронирование и у него есть 12 часов, чтобы на него отреагировать. Если владелец не реагирует на запрос или отклоняет его, то запрос на бронирование автоматически отменяется.

После подтверждения брони пользователь получает push-уведомление. У пользователя есть 12 часов, чтобы оплатить сумму предоплаты и таким образом подтвердить бронирование. Пользователь может оплатить бронирование с помощью банковской карты.

Во время процесса бронирования жилье недоступно для бронирования другим пользователями (на выбранные даты). В случае отмены бронирования жилье снова становится доступно другим пользователям сервиса.

---

### Регистрация/Авторизация

Пользователь может зарегистрироваться в приложении с помощью номера телефона. На указанный номер приходит SMS с кодом подтверждения.

После регистрации предлагаем пользователю заполнить информацию для бронирований:
- Фамилию
- Имя
- Фото
- Возраст (минимальный возраст 18 лет)
- E-mail

Пользователь может пропустить этот шаг и заполнить информацию в профиле.

---

### Бронирования

Отображаются бронирования пользователя. Пользователь может перейти к бронированию. Отображается информация о бронировании:
- информация об объекте
- адрес объекта (есть возможность посмотреть адрес в приложении с картами на телефоне)
- дополнительные параметры:
  - дата и время заселения/выселения
  - инструкции, как попасть внутрь
  - правила проживания
  - дополнительная информация

Отображаются контакты владельца. Можно написать в чат владельцу.

Можно отменить бронирование:
- запрос на отмену отправляется владельцу
- предоплата, оплаченная через сервис, не возвращается

Можно изменить бронирование. При изменении бронирование повторно отправляется на подтверждение владельцу.

Можно пожаловаться на бронирование:
- предлагаем пользователю решить разногласия полюбовно с владельцем жилья
- пользователь может обратиться в поддержку
- в чат с владельцем жилья подключается администратор для решения возникшего вопроса

Есть возможность перейти в архив бронирований. Пользователь может подать жалобу на жилье/владельца (отправить текст, фото — заявка попадает на рассмотрение в техподдержку).

---

### Избранное

Отображается список объявлений, добавленных в избранное. Пользователь может перейти в объявление (отображается вся информация). Пользователь может удалить объявление из избранных.

---

### Отзывы

После окончания аренды предлагаем пользователю оставить отзыв о жилье. Можно поставить оценку по 5-бальной шкале. Можно добавить комментарий.

---

### Чат

Отображаются чаты, каждый чат привязан к конкретному жилью. Можно перейти в чат. В чате можно отправлять текстовые сообщения и изображения. При получении сообщения в чате пользователь получает push-уведомление.

---

### Профиль

Можно редактировать информацию о себе. Можно написать в поддержку. Можно выйти из аккаунта. Можно удалить аккаунт.

---

## Веб-сайт для владельцев жилья

### Регистрация/Авторизация

Пользователь может зарегистрироваться с помощью email + пароль. Есть функционал "забыли пароль".

После регистрации пользователь заполняет информацию о себе:
- Фото
- Имя (обязательно)
- Телефон (обязательно)

Предлагаем пользователю указать подробную информацию о себе (можно заполнить позже в профиле). Пользователь получает доступ к функционалу сервиса.

---

### Создание объявления

Пользователь может создать объявление:
- Указывает название объявления
- Указывает адрес
- Вид жилья
- Добавляет фотографии (минимум 6 шт)
- Добавляет описание жилья

Пользователь указывает дополнительные параметры:
- Тип размещения
- Удобства
- Доп. характеристики
- На какое количество гостей рассчитано
- Можно с животными или нет
- Есть ли возможность отмены бронирования

Заполняет в свободной форме разделы:
- инструкции, как попасть внутрь
- правила проживания
- дополнительная информация

Указывает время заселения и выселения. Указывает стоимость для клиента (пользователю отображается стоимость с учетом комиссии сервиса):
- за аренду посуточно
- за аренду на 14 дней и более (стоимость за сутки)
- за аренду на 28 дней и более (стоимость за сутки)

Пользователь может отправить объявление на модерацию администратору. После прохождения модерации объявление публикуется. Пользователь получает push-уведомление.

---

### Мои объявления

Отображается список объявлений. Пользователь может редактировать объявление (объявление повторно отправляется на модерацию). Пользователь может перейти в календарь бронирования. Пользователь может снять объявление с публикации. Снятые с публикации объявления попадают в архив. Можно повторно опубликовать объявление.

---

### Управление бронированиями

Отображаются заявки на бронирования от клиентов. При получении нового бронирования пользователь получает SMS-оповещение на телефон.

Пользователь может подтвердить заявку. Клиенту дается 12 часов, чтобы внести предоплату. Пользователь видит, оплатил ли клиент предоплату и сколько осталось времени. Пользователь может отклонить заявку.

Отображаются контакты клиента. Можно написать клиенту в чат.

После окончания бронирования предлагаем оставить отзыв о клиенте: поставить оценку по 5-бальной шкале, добавить комментарий.

У каждого объекта (объявления) есть свой календарь бронирования:
- В календаре отмечены даты, когда жилье занято гостями (можно перейти к гостю, который забронировал выбранные дни)
- В календаре отмечены даты, когда жилье свободно и когда жилье недоступно для съема

Пользователь может редактировать занятость в выбранный день. Владелец жилья может в одностороннем порядке отменить бронирование пользователя и освободить слот в календаре.

Если клиент отправил запрос на отмену бронирования или изменение сроков бронирования:
- Пользователь может согласовать изменение сроков бронирования
- Пользователь может согласовать отмену бронирования
- Пользователь может отказать в изменении сроков бронирования или в отмене бронирования

Клиент получает push-уведомление с информацией о решении владельца.

---

### Чат

Отображаются чаты с клиентами. В чате можно отправлять текстовые сообщения и изображения. При получении сообщения клиент получит push-уведомление.

---

### Профиль

Пользователь может редактировать информацию о себе. Отображается сумма, доступная для вывода: пользователь может запросить вывод средств. На внутренний баланс пользователя деньги зачисляются после окончания аренды и за вычетом комиссии сервиса. Пользователь может написать в поддержку. Можно выйти из аккаунта.

---

## Веб-сайт для администратора

### Регистрация/Авторизация

Для доступа к функционалу сервиса обязательно требуется регистрация. Пользователь может зарегистрироваться/авторизироваться с помощью email + пароль.

После регистрации сотрудник должен заполнить: имя, фото. После регистрации сотрудник попадает на проверку администратору и не получает доступ.

---

### Администратор. Сотрудники

Отображаются сотрудники и их роли. Администратор может изменить роль сотрудника. Администратор может заблокировать доступ сотрудника. Администратор может разблокировать доступ сотрудника.

Отображаются заявки на регистрацию от пользователей. Администратор может принять заявку (выбирает задачи, который этот пользователь может выполнять). Администратор может отклонить заявку.

---

### Модерация объявлений и пользователей

Отображаются заявки на публикацию объявления. Отображается информация, указанная в объявлении. Администратор может отклонить публикацию (указывает причину отказа). Администратор может одобрить публикацию.

Отображается список опубликованных объявлений в приложении. Администратор может фильтровать объявления. Администратор может снять объявление с публикации. Администратор может перейти к аккаунту владельца объявления. Администратор может заблокировать (забанить) владельца.

---

### Техподдержка

Отображается список обращений и жалоб в поддержку. Если жалоба поступила на конкретное бронирование, то администратор подключается к чату по данному бронированию.

Администратор может вернуть деньги клиенту (указывает сумму, которую вернуть). В чате можно отправлять: текстовые сообщения, изображения, файлы.

---

### Профиль

Можно изменить информацию о себе: имя, фото. Можно выйти из аккаунта.

---
