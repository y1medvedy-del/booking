# Git шпаргалка

## Главное правило

Перед работой:

```bash
git checkout main   # перейти на главную ветку
git pull            # скачать свежие изменения с GitHub
```

После работы:

```bash
git add .                           # добавить все изменения в коммит
git commit -m "что сделал"         # сохранить изменения локально с комментарием
git push                            # отправить изменения на GitHub
```

На другом устройстве перед продолжением:

```bash
git checkout main   # перейти на главную ветку
git pull            # подтянуть последние изменения
```

## Если работаешь без веток

```bash
git checkout main                  # перейти на main
git pull                           # забрать свежую версию проекта
git add .                          # подготовить изменения
git commit -m "что сделал"        # сохранить изменения
git push                           # отправить на GitHub
```

## Если делаешь отдельную ветку

Создать ветку:

```bash
git checkout main                  # перейти на main
git pull                           # обновить main
git checkout -b feature-name       # создать новую ветку и сразу перейти в нее
```

Сохранить ветку на GitHub:

```bash
git add .                          # добавить изменения
git commit -m "что сделал"        # сохранить изменения
git push -u origin feature-name    # отправить ветку на GitHub
```

Объединить в `main`:

```bash
git checkout main                  # вернуться в main
git pull                           # обновить main
git merge feature-name             # влить ветку в main
git push                           # отправить обновленный main на GitHub
```

## Полезные команды

```bash
git status         # показать что изменено
git branch         # показать список веток
git checkout main  # перейти на main
git pull           # скачать изменения
git push           # отправить изменения
```

## Чтобы не путаться

- Всегда `git pull` перед началом работы.
- Всегда `git push` в конце работы.
- Не работай на двух устройствах одновременно в одних и тех же файлах.
- Для больших задач лучше делать отдельную ветку.
